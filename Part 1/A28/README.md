For this activity, I implemented a terminal-based password manager built using Python. It can be accessed on [GitHub](https://github.com/saacutter/pw-manager), or a copy of the important files have been provided as a subdirectory in this repository. 

This is obviously a security solution because of the nature of password managers. The backend database is just a SQLite3 database which, while not normally secure, has any senstive field encrypted. In the case of user master passwords, these are hashed using the SHA-256 algorithm and interfaced with the `passlib` Python library. This is what made the most sense for this purpose, as the master password isn't intended to be reversible (and hence a one-way hash function is fine and reduces the likelihood of a successful attack). However, a separate system had to be used for stored user passwords. These have to be reversible (otherwise the purpose of it managing passwords is moot), so another algorithm had to be used. This was accomplished with the `cryptography` library using the Fernet encryption system. This is a symmetric encryption system, meaning that the same key is used to encrypt and decrypt data. A salt is used to reduce the likelihood of attack (and stored so the same key can be used every login), and sensitive information is hashed using this key. Whenever this data is requested by an authorised individual, it is then decrypted using this same key so it is visible to them.
