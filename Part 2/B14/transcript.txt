I'll start anyway

So hashing is used to convert strings of arbitrary length into fixed length strings. There's a few different algorithms for this, but the main ones are probably MD5, SHA-1, and SHA-256. These are usually designed to be one way (so unlike encryption they can't be reversed), and it should be infeasible to find two hashes that have the same resultant hash value (which includes it being infeasible to find another hash that matches another and being infeasible to find two strings that hash to the same thing).

It gets interesting when talking about passwords because it's likely that two people will have the same password (this is basically the birthday problem because there are a finite amount of possible passwords). This is a problem because if the hashing algorithm is known, then an attacker could just hash the most common passwords (like password, password1, etc) and can then log into any account using those hashes. Therefore, another security mechanism is needed to make passwords more secure - salting. Salting is basically adding a random value to the end of the password (so password + salt) and then hashing that. Since hashes will significantly change with even a single bit difference, this means that you can have the same password hashed with different salts resulting in different hashes, making these attacks much harder to use.

Now you might be asking, how does finding if two password hashes work if it uses a random salt? Well, the password keeps a copy of the salt. Because hashes are irreversible, these can be stored with the password (on Linux it's stored like $salt$algorithm$hash). This means that whenever the user signs in, the salt is appended to the entered password and the resulting hash is compared

There are a few different types of hashing algorithms, like insecure ones (used for purposes like checksums to make sure files haven't been tampered with because they have been cracked and therefore are insecure like MD5), secure hashes (which are stronger algorithms which are used for more secure purposes like SHA-256), and password hashing algorithms (which are the strongest algorithms used for passwords specifically because they have the strongest protection like bcrypt).
